T - Task 工作指引
C - Context 任務脈絡
R - Reference 參照點
E - Evaluate 評估標準
I - Iterate 迭代

T、C、R 是提詞的「內容結構」。
E、I 是優化提詞的「行動方法」。

▋ T - Task 工作指引

告訴 AI 它的角色、任務、限制、風格。

→ 角色：指定 AI 的身份。
  例：「你是一位資深產品分析師」。

→ 任務：說明要完成的具體工作。
  例：「分析使用者回饋，提煉前三大抱怨的核心問題」。

→ 限制：定義不該做的事。

→ 風格：指定溝通語氣。

▋ C - Context 任務脈絡

提供任務背後的「原因」，比工作指引更重要。高品質的產出，來自高品質的脈絡，而非複雜的指令。「脈絡工程」的核心，是挑選正確的資訊餵給 AI。

→ 脈絡心法：從最簡單的版本開始，逐步增加細節。

→ 脈絡包含：

  → 我（使用者）是誰：例：「我是一位產品經理」。
  → 目標理想值：期望的最終成果。例：「為下一次產品迭代提供清晰的方向」。
  → 要解決的問題：目前面臨的挑戰。
  → 問題的重要性：為何需要解決。例：「這次分析將決定未來三個月的開發重點」。
  → 我的資源：擁有的資料或工具。
  → 環境限制：客觀上無法做到的事。
  → 需要的協助：明確指出 AI 的任務。
  → 產出對象：內容要給誰看。
  → 產出範例：提供具體的成功範例。
  → 使用架構：指定 AI 應遵循的思考框架。

▋ R - Reference 參照點

提供「模具」或「導軌」，引導 AI 產出符合預期的形狀與方向。
參考架構包含四種：

→ 步驟：將複雜任務拆解，引導 AI 依序完成，能顯著提升精準度。
  例：「第一步：通讀所有回饋；第二步：分類；第三步：排序；第四步：總結。」

→ 格式：指定輸出格式，方便後續使用。
  例：要求用 Markdown 表格或 JSON 格式輸出。

→ 範例：提供標準範例，讓 AI 模仿其模式、風格、格式。這是最有效的引導方式。

→ 架構：提供思考框架的關鍵字，讓 AI 套用特定方法論。
  例：要求使用「批判性思維」、「OODA」、「飛輪思維」進行分析。

▋ E - Evaluate 評估標準

提詞是一個動態過程，評估標準用來判定產出是否達標，以及流程何時可以結束。

你心中要有一把尺，清楚分辨產出的好壞。這把尺反映了你的領域知識：你理解越深入，設定的標準越高，寫出的提詞就越強大。

如果沒有評估標準，就像上了國道卻不知道目的地。

▋ I - Iterate 迭代

好的提詞，通常需要反覆修改。

→ 從最簡單的版本開始，一輪一輪打磨。
→ 每次只修改一小部分，才能釐清問題所在。
→ 不要一次寫滿所有指令，這會讓除錯變得極其困難。
