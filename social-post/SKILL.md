# social-post

## GATE：強制對焦（不可跳過）

收到任務後，第一步必須：
1. 一句話複述「我理解你要處理哪篇素材、要發到哪裡」
2. 判斷目前階段：前期準備 / 產出 / 優化迭代
3. 說明這輪會先走正式選項對焦：讀者 → 痛點 → 收穫 → 場景 → 公式/寫法
4. 問「方向對嗎？」→ 用戶說 OK 才往下

## 流程

### 步驟 1：前期準備

Read `knowledge/workflow.md` 與 `references/axis_check.md` 執行前期準備。先收內容，再逐題完成正式選項對焦：讀者 → 痛點 → 收穫 → 場景 → 公式/寫法。完成後還要補上人物路徑線，以及正文骨架（`SCQA + 3-5 個主論點`）。若當前 session 支援正式互動式選項，優先使用；若不支援，退回同構的文字選項流程。

> **強制停止點：軸心卡與公式方向確認後，必須等用戶確認才能開始正式寫稿。**

### 步驟 1.5：長文深度寫作（Mode C / 個人故事 / 書評觀點）

若判斷為長文類型（Mode C 深度反思、個人故事型、書評觀點型），改走 `knowledge/long_form_writing_sop.md` 的完整 8 Phase 流程。短貼文（Mode A/B）跳過此步驟。

### 步驟 2：文章產出

Read `knowledge/writing_marketing.md` 執行文案生成。生成時必須同時套用 `style_profile.md`、`references/rules.md`、`references/formulas.md`，先輸出「草稿前摘要」，再生成草稿並寫入 `drafts/`，格式遵守 `TEMPLATES.md`。前言必須先交代問題、讀者、結果與合理性；若會引用某個人，還要先交代他是誰、憑什麼值得聽、這件事為什麼和讀者有關。第一屏預設先打核心句，再補背景；診斷型長文預設先收成 4 個主論點。初稿完成後，預設還要再跑一次「文章收斂」：刪掉冗句與重複鋪陳，保留人味、細節與作者存在感；若有收斂版，預設以收斂版作為發文候選主稿。

**寫作限制（硬規則）：**
- 禁止使用破折號（——）
- 「不是 X，而是 Y」句式禁止重複使用超過 1 次，替代句式見 `Prompt/12-句式替換-不是而是.md`（與其說是/只是表面才是關鍵/除了更要/沒有只有/不只更要/比起更重要的是）
- 禁止大陸用語，如「火了」「普及了」「爆了」等；改用臺灣口語或白話描述
- 每段結尾需有一句話鉤子，讓讀者想往下看（萬能寫作法：重點 → 展開 → 一句話總結）
- 長文結尾加「重點回顧」段落（見 `Prompt/7-重點回顧.md`：3-4 句最有啟發性的重點，一句話內保持具體）
- 長文最後用金句收尾（見 `Prompt/8-文章收尾.md`：從對比/具象化/搜詞法/ABBA/ABAC 五種模板中選用）

**F9 Public L-taking（我錯了反轉文）特定規則：**
- 前言必須爆點前置：L-taking 核心時刻（「我錯了/我是 X」）必須在前言主體前 50 字內出現，不能先鋪陳再到達
- 個人故事整合進前言，不另起「故事段」放在論點之後；論點內可有各自的小故事支撐
- 適合使用 stanza 短句格式（每 2-4 字一行，段落間保留空白行），製造情感節奏與停頓感
- 前言結構：個人 L-taking 故事 → 拉出普遍性診斷 → 列點預告論點
- 收尾結構：濃縮四件事 → 核心金句回顧 → 個人轉化弧 → CTA
- CTA 格式：「[資源說明]，[行動方式，如連結放留言]。歡迎來。」語氣輕，不強推

### 步驟 3：封面產出

Read `knowledge/image_pipeline.md` 執行封面流程。文案定稿並拿到使用者確認後，再進入封面流程；封面預設產出 2 張 `16:9 / 1920x1080` 變體，統一存到 `photo/`，檔名用主標題加流水號。開始做圖前，必須先看 `photo/` 內至少 2 張既有成品，優先沿用同一家族的構圖、質感與文字密度；若這輪工具只能產出 prompt、SVG、HTML 或其他中間稿，不能把它們當成封面完成品交付。

> **強制停止點：正式發佈前，必須把文案與封面變體交給用戶確認；沒拿到明確「確認」字眼就不能發。**

### 步驟 4：發佈與記錄

Read `references/generate_and_publish.md` 執行平台發佈。發完後更新 `content_plan.md`，記錄今天題材、公式、平台與待追蹤戰績。

### 步驟 5：優化與迭代

Read `knowledge/iteration.md` 執行第二次迭代。發文後依 48-72h 數據判讀內容表現，更新 `content_plan.md`，再決定下一篇要延續、改寫還是換公式。

### 步驟 6：品質檢查（必須執行）

Read `TEMPLATES.md` 與 `knowledge/workflow.md`，逐項檢查：
- 是否已依序完成逐題選項對焦
- 軸心卡是否完整
- 人物路徑線與正文骨架是否完整
- 草稿前摘要是否存在且與軸心卡一致
- 草稿是否符合 Fish 語氣與選定公式
- 前言是否已交代合理性，沒有陌生人物突然冒出來
- 草稿是否已完成文章收斂，讀起來比原始展開版更順、更短、更少贅述
- 草稿是否有真實場景、主觀判斷與作者存在感，不只是整理資訊
- 若有進封面流程，是否已產出 2 張封面變體
- 若有進封面流程，是否已先比對 `photo/` 內至少 2 張既有封面，沒有風格漂移
- 命名、資料夾、尺寸是否正確
- 是否交付真實可用的 raster 圖檔（`.png` / `.jpg`），不是 prompt、佔位稿或中間檔
- 若進入迭代，是否已根據數據更新 `content_plan.md`

**文章評估（長文必跑，見 `Prompt/11-文章評估.md`）：**
逐項診斷 8 個維度，找出最弱的 1-2 項再優化：
1. 標題穿透力：3 秒內知道給什麼、是否反常識
2. 開場鉤子強度：前 50 字是否出現爆點（F9 必查）
3. 副標題切角：診斷式、不重複、語氣夠犀利
4. 內容結構完整性：萬能寫作法三段式、案例有人名/數字/場景
5. 案例衝擊力：夠極端、夠具體、能讓讀者看見畫面
6. 語氣與態度：診斷式非教學式、平等非高高在上
7. 金句密度：每 2-3 段至少一句可截圖金句
8. 收尾完整性：重點回顧 → 核心訊息 → 行動誘因
