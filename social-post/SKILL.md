# social-post

## GATE：強制對焦（不可跳過）

收到任務後，第一步必須：
1. 一句話複述「我理解你要處理哪篇素材、要發到哪裡」
2. 判斷目前階段：前期準備 / 產出 / 優化迭代
3. 說明這輪會先走正式選項對焦：讀者 → 痛點 → 收穫 → 場景 → 公式/寫法
4. 問「方向對嗎？」→ 用戶說 OK 才往下

---

## 流程

### 步驟 1：前期準備

Read `knowledge/workflow.md` 與 `references/axis_check.md` 執行前期準備。先收素材，再逐題完成正式選項對焦：讀者 → 痛點 → 收穫 → 場景 → 公式/寫法。若當前 session 支援正式互動式選項，優先使用；若不支援，退回同構的文字選項流程。

> **強制停止點：軸心卡與公式方向確認後，必須等用戶確認才能開始正式寫稿。**

### 步驟 2：文章產出

Read `knowledge/writing_marketing.md` 執行文案生成。生成時必須同時套用 `style_profile.md`、`references/rules.md`、`references/formulas.md`，先輸出「草稿前摘要」，再生成草稿並寫入 `drafts/`，格式遵守 `TEMPLATES.md`。初稿完成後，預設還要再跑一次「文章收斂」：刪掉冗句與重複鋪陳，保留人味、細節與作者存在感。

### 步驟 3：封面產出

Read `knowledge/image_pipeline.md` 執行封面流程。文案定稿並拿到使用者確認後，再進入封面流程；封面預設產出 2 張 `16:9 / 1920x1080` 變體，統一存到 `photo/`，檔名用主標題加流水號。

> **強制停止點：正式發佈前，必須把文案與封面變體交給用戶確認；沒拿到明確「確認」字眼就不能發。**

### 步驟 4：發佈與記錄

Read `references/generate_and_publish.md` 執行平台發佈。發完後更新 `content_plan.md`，記錄今天題材、公式、平台與待追蹤戰績。

### 步驟 5：優化與迭代

Read `knowledge/iteration.md` 執行第二次迭代。發文後依 48-72h 數據判讀內容表現，更新 `content_plan.md`，再決定下一篇要延續、改寫還是換公式。

### 步驟 6：品質檢查（必須執行）

Read `TEMPLATES.md` 與 `knowledge/workflow.md`，逐項檢查：
- 是否已依序完成逐題選項對焦
- 軸心卡是否完整
- 草稿前摘要是否存在且與軸心卡一致
- 草稿是否符合 Fish 語氣與選定公式
- 草稿是否已完成文章收斂，讀起來比原始展開版更順、更短、更少贅述
- 草稿是否有真實場景、主觀判斷與作者存在感，不只是整理資訊
- 若有進封面流程，是否已產出 2 張封面變體
- 命名、資料夾、尺寸是否正確
- 若進入迭代，是否已根據數據更新 `content_plan.md`
