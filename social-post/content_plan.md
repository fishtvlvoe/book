## 本輪策略摘要

- 品牌主軸不是「我懂很多」，而是「我真的一直在做，做完還公開」。
- 名單收集不走硬賣，優先用 `留言關鍵詞`、`我整理公開版`、`想要我再發` 這類低摩擦 CTA。
- 輕重節奏採交錯：**進度 / 觀點 / 可領取 / 宣言 / 再回到進度**，避免每天都像在賣東西。

## 即時校正（2026-06-18）

- 這次改看舊文高互動樣本，不用最近幾天的低曝光貼文當判準。內容庫用 `總期間 + 留言數排序` 後，前段樣本很明確：
  - `2025-11-10`：`史蒂夫·賈伯斯建立 App Store 花了 10 年，伊隆·馬斯克再 5 年可能讓它消失`，`714 留言 / 2,812 分享`
  - `2025-10-15`：`前美國特勤局特工的 9 個生存法則`，`79 留言 / 4,178 分享`
  - `2025-10-13`：`賣掉房子，我住在特斯拉裡一年`，`160 留言 / 432 分享`
  - `2025-12-17`：`黃仁勳說「我不祝你們一帆風順，我祝你們痛苦不堪」`，`67 留言 / 1,375 分享`
  - `2025-12-16`：`為什麼你能接受 AI 寫程式，卻不能接受 AI 寫文章？`，`42 留言 / 132 分享`
- 這幾篇共通點不是「最近在忙什麼」，而是：
  - 一開頭就打讀者的世界觀，不是先講你的近況
  - 借名人、極端案例、或兩派對立，轉成一般人會代入的工作焦慮 / 身分焦慮 / 生存焦慮
  - 句子本身要有轉傳價值，讀者會想丟給朋友說「你看，這句有意思」
- 所以下一篇最適合發的，不是純 workflow 解釋文，也不是再一篇整理卡。
- 最適合的是：**用 AI / 工作 / 專業焦慮當戰場，直接給第三立場**。
- 建議公式：`F10 兩派都錯`
- 建議題目：`AI 派說工程師要被取代，老工程師說 AI 寫的都是垃圾。兩邊都錯。`
- 建議 CTA：留言 `拆工`，下一篇再把「哪些工作該交給 AI，哪些一定要自己扛」寫白。正文不放外部連結。

## 為什麼這樣排

- **品牌**：F25a / F2 / F5 / F8 / F10 / F22 / F25c 會把你定位成「真的在做，而且會公開方法的人」。
- **名單**：F21 / F27 / F12 讓你有自然理由叫人留言、索取、回來看結果。
- **節奏**：每 2-3 天才放一次明確收名單題，避免整排都像 lead magnet。
